# 每日复盘模块产品需求文档

## 1. 背景与目标

韭菜盘 / JCP AI 当前已经具备行情、K 线、盘口、F10、龙虎榜、热点舆情、研报、多 Agent 讨论、策略管理、记忆系统等股票分析能力。用户在盘中和盘后会得到大量行情数据、AI 观点、交易计划和复盘结论，但这些内容目前主要沉淀在股票会话、记忆文件或临时交互中，缺少一个面向“交易日复盘”的长期记录与比较模块。

每日复盘模块的目标是提供一个本地化、Markdown 化、可持续积累的复盘工作台，让用户可以在每个交易日记录市场表现、持仓变化、交易决策、情绪变化、错误原因和改进计划，并支持单日展示、多日对比、总复盘总结、模板化录入和图片自动归档。

## 2. 当前系统基础

### 2.1 已实现能力

- 股票行情：支持自选股、实时行情、分时、日 K、周 K、月 K、五档盘口、大盘指数和市场状态识别。
- 图表分析：前端使用 Lightweight Charts 展示 K 线，并支持常用技术指标配置。
- AI 讨论室：支持用户提问、指定专家 Agent、多 Agent 串行讨论、流式进度、失败重试、取消会议、引用回复和消息清空。
- 策略系统：支持内置策略和自定义策略，每个策略可配置多个专家角色、工具权限、MCP 服务和独立 AI 模型。
- AI 配置：支持 OpenAI 兼容接口、Gemini、Vertex AI、Anthropic、Responses API、代理配置和连接测试。
- 记忆系统：按股票隔离长期记忆，支持关键事实、最近讨论轮次、摘要压缩和相关性检索。
- 数据增强：包含 F10 全景、估值、财务、机构、行业、公告、研报、财联社快讯、热点平台、龙虎榜、板块资金流、盘口异动等。
- 扩展能力：支持 MCP 管理器和 OpenClaw HTTP 服务。
- 桌面体验：支持无边框窗口、主题切换、面板拖拽布局持久化和自动更新。

### 2.2 技术栈与约束

- 桌面框架：Wails v2，Go 后端绑定给 React 前端调用。
- 后端语言：Go 1.24。
- 前端框架：React 18 + TypeScript + Vite。
- UI 与样式：TailwindCSS、Lucide React 图标、自定义主题上下文。
- Markdown 渲染：现有 AI 回复使用 `markstream-react`。
- 本地存储：当前配置、策略、会话、记忆等主要保存在应用数据目录 `paths.GetDataDir()` 下。
- SQLite：`go.mod` 已包含 `github.com/glebarez/go-sqlite` 和 `modernc.org/sqlite` 间接依赖，可在需要索引、检索和关联查询时使用。

## 3. 产品定位

每日复盘模块定位为“交易日知识沉淀中心”，不是替代 AI 讨论室，而是把盘后总结、交易行为、截图、关键结论和长期经验沉淀到结构化文章中。

核心价值：

- 把每日交易复盘固化为 Markdown 文件，便于迁移、备份和版本管理。
- 把图片统一归档到 pictures 目录，避免剪贴板图片、网络图片和文章内容分散。
- 用模板降低每日输入成本，提升复盘一致性。
- 用多日对比发现重复错误、策略有效性和市场环境变化。
- 用总复盘总结文章持续积累经验，形成个人交易手册。

## 4. 用户角色与使用场景

### 4.1 主要用户

- 主动交易者：每天需要记录市场、持仓、买卖点和交易情绪。
- 中长期投资者：需要周期性记录重要结论、估值变化和跟踪事项。
- AI 辅助分析用户：希望把 AI 讨论室里的观点整理成复盘文章。

### 4.2 核心场景

- 盘后新建当日复盘，选择一个模板，填写今日市场、操作、错误、改进计划。
- 在文章中粘贴截图，系统自动保存图片到 pictures 目录，并把 Markdown 图片链接改为本地相对路径。
- 在文章中粘贴网络图片 URL，系统自动下载图片，保存到 pictures 目录，并替换为本地相对路径。
- 查看某一天的复盘文章，阅读当日总结、图片和记录。
- 选择多个日期进行复盘对比，横向比较市场环境、操作质量、盈亏、情绪、错误类型和计划执行情况。
- 长期维护一篇总复盘总结文章，把高频错误、有效策略、交易纪律和经验清单持续沉淀进去。
- 自定义复盘模板，新建每日复盘时选择模板，并在模板基础上编辑保存。

## 5. 功能需求

### 5.1 复盘入口

在主界面顶部工具区新增“每日复盘”入口，建议使用 `BookOpen`、`NotebookPen` 或 `FileText` 图标。点击后打开复盘工作台弹窗或右侧独立面板。

首期建议采用全屏弹窗方式，避免破坏现有左中右股票分析布局；后续可扩展为独立主导航页面。

### 5.2 文章列表

文章列表展示所有每日复盘文章和总复盘总结文章。

列表字段：

- 日期：复盘归属日期，格式 `YYYY-MM-DD`。
- 标题：默认 `YYYY-MM-DD 每日复盘`，支持用户修改。
- 模板名称：文章创建时使用的模板。
- 标签：如 `大亏`、`纪律问题`、`趋势行情`、`情绪交易`。
- 关联股票：文章中主动选择或自动识别的股票代码列表。
- 图片数量：文章引用的本地图片数量。
- 更新时间：文件或索引的最近更新时间。
- 摘要：从 Front Matter 的 `summary` 或正文前 N 字自动生成。

列表能力：

- 按日期倒序展示。
- 支持按日期范围筛选。
- 支持标题、正文、标签、股票代码搜索。
- 支持只看总复盘总结。
- 支持创建、打开、删除、重命名每日复盘。
- 删除文章时默认不删除图片，只标记孤立图片，后续提供图片清理能力。

### 5.3 新建每日复盘

用户点击“新建复盘”后进入创建流程：

1. 默认日期为当前日期，允许选择任意日期。
2. 如果该日期已存在复盘文章，则提示打开已有文章或另存为副本。
3. 选择复盘模板。
4. 根据模板生成 Markdown 初始内容。
5. 自动写入 Front Matter 元数据。
6. 保存为 `articles/YYYY-MM-DD.md`。

默认 Front Matter：

```yaml
---
type: daily_review
date: 2026-04-27
title: 2026-04-27 每日复盘
templateId: default-daily
templateName: 标准每日复盘
tags: []
stocks: []
profitLoss: 0
emotion: ""
disciplineScore: null
createdAt: 2026-04-27T15:30:00+08:00
updatedAt: 2026-04-27T15:30:00+08:00
---
```

### 5.4 Markdown 编辑与展示

编辑器要求：

- 支持 Markdown 原文编辑。
- 支持预览模式。
- 支持编辑 / 预览左右分栏。
- 支持保存快捷键 `Ctrl+S`。
- 支持未保存离开提示。
- 支持自动保存，默认 30 秒防抖保存一次。
- 支持标题、正文、Front Matter 同步更新。

展示要求：

- 单日复盘展示时渲染 Markdown、图片、表格、代码块、引用和任务列表。
- 图片使用本地相对路径加载。
- 文章缺失图片时展示占位提示，不阻断全文阅读。
- 支持从文章跳转到关联股票分析页面。

实现建议：

- 首期可复用 `markstream-react` 作为预览渲染器。
- 编辑器首期可用 `<textarea>` 实现，后续再引入 CodeMirror / Monaco / Milkdown 等增强编辑体验。
- 图片粘贴和拖拽上传应在编辑器层处理，然后调用后端保存图片。

### 5.5 图片归档

图片目录：

```text
<dataDir>/
  articles/
    2026-04-27.md
    复盘总结.md
  pictures/
    2026/
      04/
        27/
          20260427-153012-a1b2c3.png
          20260427-154855-d4e5f6.jpg
  review_templates/
    default-daily.md
    short-daily.md
  review.db
```

粘贴图片：

- 用户从剪贴板粘贴图片到编辑器。
- 前端读取剪贴板中的 `image/png`、`image/jpeg`、`image/webp`。
- 前端将图片以 base64 或二进制传给后端。
- 后端生成文件名，保存到 `pictures/YYYY/MM/DD/`。
- 后端返回 Markdown 相对路径，例如 `../pictures/2026/04/27/20260427-153012-a1b2c3.png`。
- 前端在光标位置插入 `![图片说明](../pictures/2026/04/27/20260427-153012-a1b2c3.png)`。

网络图片：

- 用户粘贴 Markdown 图片链接 `![alt](https://example.com/a.png)` 或直接粘贴图片 URL。
- 系统识别网络图片 URL。
- 调用后端下载图片。
- 后端校验 Content-Type、扩展名和大小限制。
- 保存到 pictures 目录。
- 替换为本地相对路径。

下载规则：

- 支持 `http` 和 `https`。
- 默认最大单图 10 MB，可在配置中调整。
- 下载超时默认 15 秒。
- 文件名使用日期、时间和内容 hash，避免重复。
- 相同 URL 或相同内容 hash 可复用已有图片。

### 5.6 单日复盘展示

单日展示页包含：

- 日期切换器：上一交易日、下一交易日、返回列表。
- 文章标题和元信息。
- Markdown 正文渲染。
- 图片查看。
- 关联股票列表。
- 标签列表。
- 编辑按钮。
- 复制 Markdown 按钮。

增强能力：

- 支持从当前日期拉取当日市场状态、大盘指数、龙虎榜、热点、已选自选股表现，作为辅助信息区展示。
- 支持把 AI 讨论室选中的消息复制或追加到当日复盘。

### 5.7 多日复盘对比

多日对比入口：

- 在文章列表中多选日期后点击“对比”。
- 在对比页面选择日期范围，例如最近 5 个交易日、最近 20 个交易日、自定义日期。

对比维度：

- 每日标题、摘要、标签。
- 盈亏数据 `profitLoss`。
- 纪律评分 `disciplineScore`。
- 情绪字段 `emotion`。
- 关联股票。
- 错误类型。
- 明日计划。
- 文章正文关键段落。

推荐模板化字段：

```yaml
reviewFields:
  market: 今日市场
  actions: 今日操作
  mistakes: 错误与偏差
  lessons: 经验教训
  nextPlan: 明日计划
```

对比展示方式：

- 横向表格：日期为列，复盘字段为行。
- 纵向时间线：按日期展示摘要、标签、计划与执行。
- 标签聚合：展示一段时间内最高频错误、最高频股票、最高频情绪。

首期范围：

- 先实现横向表格和正文摘要。
- 字段提取优先读取 Front Matter 和标准 Markdown 标题。
- 无法结构化提取时显示正文前 200 字摘要。

### 5.8 总复盘总结文章

系统内置一篇总复盘总结文章：

- 文件路径：`articles/复盘总结.md`。
- 类型：`summary_review`。
- 可无限编辑和保存。
- 不绑定具体日期。
- 在文章列表中置顶展示。

默认内容建议：

```markdown
# 复盘总结

## 高频错误

## 有效策略

## 交易纪律

## 仓位管理

## 情绪管理

## 可复用检查清单
```

增强能力：

- 从多日复盘对比页一键把选中结论追加到总复盘总结。
- 支持“追加到复盘总结”操作，自动添加日期引用。
- 后续可引入 AI 汇总，把指定日期范围内的复盘提炼为经验条目。

### 5.9 自定义复盘模板

模板管理能力：

- 新建模板。
- 编辑模板。
- 删除模板。
- 设置默认模板。
- 复制模板。
- 从当前文章另存为模板。

模板存储：

- 模板正文保存为 Markdown 文件。
- 模板元数据保存到 SQLite 或模板 Front Matter。
- 路径为 `review_templates/{templateId}.md`。

模板字段：

- `id`：模板唯一 ID。
- `name`：模板名称。
- `description`：模板描述。
- `content`：Markdown 模板正文。
- `isBuiltin`：是否内置模板。
- `createdAt`：创建时间。
- `updatedAt`：更新时间。

默认内置模板：

```markdown
# {{date}} 每日复盘

## 今日市场

- 上证：
- 深成指：
- 创业板：
- 市场情绪：

## 今日操作

| 股票 | 操作 | 价格 | 仓位 | 理由 | 结果 |
| ---- | ---- | ---- | ---- | ---- | ---- |

## 持仓复盘

| 股票 | 当前判断 | 风险点 | 下一步计划 |
| ---- | ---- | ---- | ---------- |

## 错误与偏差

- 

## 做得好的地方

- 

## 情绪记录

- 

## 明日计划

- 
```

模板变量：

- `{{date}}`：复盘日期。
- `{{weekday}}`：星期。
- `{{title}}`：默认标题。
- `{{marketStatus}}`：市场状态。
- `{{selectedStocks}}`：当前自选股列表。

首期至少支持 `{{date}}`、`{{weekday}}`、`{{title}}`。

## 6. 数据设计

### 6.1 存储原则

- Markdown 文件是文章内容真源。
- SQLite 只保存索引、元数据、模板索引和图片引用关系。
- 即使 SQLite 损坏，也可以通过扫描 articles 目录恢复文章列表。
- 图片文件保存在 pictures 目录，文章中只写相对路径。

### 6.2 文件结构

```text
<dataDir>/
  articles/
    2026-04-27.md
    2026-04-28.md
    复盘总结.md
  pictures/
    2026/
      04/
        27/
  review_templates/
    default-daily.md
  review.db
```

### 6.3 SQLite 表设计

`review_articles`：

```sql
CREATE TABLE review_articles (
  id TEXT PRIMARY KEY,
  type TEXT NOT NULL,
  date TEXT,
  title TEXT NOT NULL,
  file_path TEXT NOT NULL UNIQUE,
  template_id TEXT,
  template_name TEXT,
  summary TEXT,
  tags TEXT NOT NULL DEFAULT '[]',
  stocks TEXT NOT NULL DEFAULT '[]',
  profit_loss REAL DEFAULT 0,
  emotion TEXT DEFAULT '',
  discipline_score INTEGER,
  image_count INTEGER DEFAULT 0,
  created_at INTEGER NOT NULL,
  updated_at INTEGER NOT NULL
);
```

`review_templates`：

```sql
CREATE TABLE review_templates (
  id TEXT PRIMARY KEY,
  name TEXT NOT NULL,
  description TEXT DEFAULT '',
  file_path TEXT NOT NULL UNIQUE,
  is_builtin INTEGER NOT NULL DEFAULT 0,
  is_default INTEGER NOT NULL DEFAULT 0,
  created_at INTEGER NOT NULL,
  updated_at INTEGER NOT NULL
);
```

`review_assets`：

```sql
CREATE TABLE review_assets (
  id TEXT PRIMARY KEY,
  article_id TEXT,
  source_type TEXT NOT NULL,
  source_url TEXT DEFAULT '',
  hash TEXT NOT NULL,
  file_path TEXT NOT NULL UNIQUE,
  mime_type TEXT NOT NULL,
  size INTEGER NOT NULL,
  created_at INTEGER NOT NULL
);
```

说明：

- `tags` 和 `stocks` 首期用 JSON 字符串保存，减少表结构复杂度。
- `file_path` 存相对 `<dataDir>` 的路径，便于迁移。
- `date` 对总复盘总结为空。

### 6.4 Front Matter 与索引同步

保存文章时：

- 解析 Front Matter。
- 更新 `review_articles`。
- 扫描 Markdown 图片引用。
- 更新 `image_count` 和 `review_assets.article_id`。

启动或手动重建索引时：

- 扫描 articles 目录。
- 读取每个 `.md` 文件。
- 解析 Front Matter。
- 重新写入 SQLite。

### 7.4 安全与稳定性要求

- 所有文章和图片路径必须限制在 `articles`、`pictures`、`review_templates` 目录内，防止路径穿越。
- 保存文章采用临时文件 + rename，避免异常中断导致文件损坏。
- 网络图片下载限制协议、大小和超时。
- 图片 MIME 类型需要校验，只允许常见图片类型。
- 删除文章前进入确认流程。
- 数据库写入失败时保留 Markdown 文件，不阻断文章保存。

## 10. 非功能需求

性能：

- 文章列表 1000 篇以内打开时间小于 500 ms。
- 单篇 Markdown 1 MB 以内编辑流畅。
- 图片粘贴保存小于 2 秒，网络图片下载受网络影响但需有进度或加载状态。

可靠性：

- Markdown 文件保存失败时必须提示用户。
- SQLite 索引失败不应导致文章丢失。
- 支持重建索引。

可迁移性：

- 用户复制 articles、pictures、review_templates 即可迁移核心内容。
- review.db 可重建。

跨平台：

- 路径处理统一使用 Go `filepath`。
- Markdown 中统一使用 `/` 作为相对路径分隔符。

## 12. 验收标准

基础验收：

- 可以从主界面打开每日复盘模块。
- 可以新建指定日期的每日复盘。
- 新建时可以选择模板。
- 文章保存为 Markdown 文件，并位于 articles 目录。
- 可以编辑和预览文章。
- 可以查看文章列表。
- 可以打开单日复盘。
- 系统存在并可编辑总复盘总结文章。

- Markdown 正文自动插入本地相对图片路径。
- 粘贴网络图片 URL 后，系统可下载到 pictures 目录并替换链接。
- 重新打开文章时图片可以正常展示。

模板验收：

- 可以查看默认模板。
- 可以新增、编辑、删除自定义模板。
- 新建每日复盘时可以选择自定义模板。

- 可以选择多个日期进入对比页。
- 可以按日期展示文章摘要、标签、股票、盈亏、情绪和纪律评分。
- 无结构化字段时能回退展示正文摘要。

存储验收：

- 删除 review.db 后，系统可以通过重建索引恢复文章列表。
- 文章文件被外部修改后，重新打开能读取最新内容。
