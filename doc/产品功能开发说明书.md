# 每日复盘模块产品功能开发说明书

## 1. 文档说明

本文档基于《每日复盘模块产品需求文档》编写，用于指导开发人员在 JCP AI 现有 Wails v2 + Go + React/TypeScript 架构中落地“每日复盘”模块。

本文档重点回答：

- 每日复盘模块首期实现哪些功能。
- 后端服务、数据模型、SQLite、文件目录如何设计。
- Wails API 如何定义。
- 前端组件、服务封装和交互流程如何实现。
- 图片粘贴、网络图片下载、Markdown 保存和索引重建如何处理。
- 开发阶段如何拆分，验收和测试如何执行。

## 2. 开发目标

### 2.1 总目标

在现有股票分析桌面端中新增“每日复盘”模块，支持用户以 Markdown 文件形式创建、编辑、存储、展示和对比交易复盘文章，并通过模板和图片自动归档降低日常复盘成本。

### 2.2 首期目标

第一版必须完成一个可闭环的本地复盘工作台：

- 从主界面打开复盘工作台。
- 自动初始化 `articles`、`pictures`、`review_templates` 和 `review.db`。
- 自动创建默认模板和总复盘总结文章。
- 支持创建指定日期的每日复盘。
- 支持选择模板创建文章。
- 支持文章列表、搜索、打开、编辑、保存、删除。
- 支持 Markdown 原文编辑和预览。
- 支持总复盘总结文章编辑。
- 支持粘贴图片保存到 pictures 目录并插入 Markdown。
- 支持网络图片下载到 pictures 目录并替换为本地 Markdown 链接。
- 支持多选每日复盘并进入基础对比视图。

### 2.3 非首期目标

以下能力可以预留接口或 UI 入口，但不要求第一版完整实现：

- AI 自动生成每日复盘草稿。
- AI 汇总多日复盘到总复盘总结。
- 自动读取交易记录并计算真实盈亏。
- 自动抓取历史指数、龙虎榜、热点并填充复盘。
- 富文本级 Markdown 编辑器。
- 图片清理器和孤立图片回收站。
- Markdown / HTML / 图片导出。

## 3. 现有系统接入点

### 3.1 后端接入点

当前后端服务聚合在 `app.go` 的 `App` 结构体中，服务通常在 `NewApp()` 中初始化，并通过 `func (a *App) MethodName(...)` 暴露给 Wails 前端。

本模块应新增：

- `internal/models/review.go`：复盘模型、请求响应结构。
- `internal/services/review_service.go`：复盘业务服务。
- `App.reviewService *services.ReviewService` 字段。
- `NewApp()` 中初始化 `reviewService := services.NewReviewService(dataDir)`。
- `app.go` 中新增 Wails API 方法。

### 3.2 前端接入点

当前主界面在 `frontend/src/App.tsx` 中组织顶部工具栏、左侧自选、中间行情图表、右侧 AI 讨论室。

本模块应新增：

- `frontend/src/services/reviewService.ts`：Wails API 封装和 TypeScript 类型。
- `frontend/src/components/ReviewDialog.tsx`：复盘工作台根组件。
- `frontend/src/components/ReviewArticleList.tsx`：文章列表。
- `frontend/src/components/ReviewEditor.tsx`：Markdown 编辑器。
- `frontend/src/components/ReviewPreview.tsx`：Markdown 预览。
- `frontend/src/components/ReviewCompare.tsx`：多日对比。
- `frontend/src/components/ReviewTemplateDialog.tsx`：模板管理。
- `App.tsx` 顶部工具栏新增入口按钮和 `showReview` 状态。

### 3.3 Markdown 渲染接入点

现有 AI 讨论室已使用 `markstream-react` 渲染 Markdown，因此复盘预览首期优先复用该依赖，不引入新的 Markdown 渲染库。

### 3.4 SQLite 接入点

`go.mod` 当前已存在 SQLite 间接依赖。实现时建议显式使用纯 Go SQLite 驱动，避免 CGO 依赖导致跨平台打包复杂。

推荐使用：

```go
import _ "modernc.org/sqlite"
```

数据库连接：

```go
sql.Open("sqlite", filepath.Join(dataDir, "review.db"))
```

如果最终选择 `github.com/glebarez/go-sqlite`，也必须保证 Wails Windows、macOS、Linux 构建不依赖额外本地库。

## 4. 功能范围拆分

### 4.1 文章管理

功能项：

- 获取文章列表。
- 创建每日复盘。
- 打开文章详情。
- 保存文章内容。
- 删除文章。
- 搜索文章。
- 按日期范围筛选。
- 置顶展示总复盘总结。

首期实现规则：

- 每个自然日期默认只允许一篇每日复盘，文件名为 `articles/YYYY-MM-DD.md`。
- 如果用户选择已存在日期，前端提示“打开已有文章”或“取消”。
- 删除文章只删除 `.md` 文件和索引，不删除 pictures 中的图片。
- 总复盘总结不可删除，只允许编辑。

### 4.2 Markdown 编辑与预览

功能项：

- Markdown 原文编辑。
- Markdown 预览。
- 编辑 / 预览 / 分栏三种模式。
- `Ctrl+S` 保存。
- 自动保存。
- 离开未保存提示。
- 保存后同步 Front Matter 和 SQLite 索引。

首期实现规则：

- 编辑器用 `<textarea>` 实现。
- 预览使用 `NodeRenderer`。
- 自动保存默认 30 秒防抖。
- 手动保存立即执行并清除 dirty 状态。
- 保存失败时保留编辑器内容并展示错误。

### 4.3 图片处理

功能项：

- 粘贴剪贴板图片。
- 拖拽图片到编辑器。
- 粘贴网络图片 URL。
- 粘贴 Markdown 网络图片语法。
- 保存图片到 pictures 目录。
- 插入或替换 Markdown 图片链接。

首期实现规则：

- 支持 `image/png`、`image/jpeg`、`image/webp`、`image/gif`。
- 单图默认最大 10 MB。
- 网络下载默认超时 15 秒。
- 图片路径按文章日期归档；总复盘总结图片使用 `pictures/summary/`。
- Markdown 内使用相对 `articles` 目录的路径，例如 `../pictures/2026/04/27/a.png`。

### 4.4 模板管理

功能项：

- 获取模板列表。
- 新建模板。
- 编辑模板。
- 删除自定义模板。
- 设置默认模板。
- 使用模板创建每日复盘。

首期实现规则：

- 内置模板不可删除。
- 默认模板必须始终存在。
- 模板内容存为 Markdown 文件。
- 模板元数据写入 SQLite。
- 模板变量首期支持 `{{date}}`、`{{weekday}}`、`{{title}}`。

### 4.5 多日对比

功能项：

- 文章列表多选。
- 进入对比视图。
- 显示日期、标题、摘要、标签、股票、盈亏、情绪、纪律评分。
- 尝试从 Markdown 标题提取关键段落。

首期实现规则：

- 最少选择 2 篇每日复盘才允许进入对比。
- 总复盘总结不参与多日对比。
- 对比结果由后端生成结构化数据，前端只负责展示。
- 字段提取失败时回退正文摘要。

## 5. 文件目录设计

所有复盘数据默认保存在 `paths.GetDataDir()` 下。

目录结构：

```text
<dataDir>/
  articles/
    2026-04-27.md
    2026-04-28.md
    复盘总结.md
  pictures/
    2026/
      04/
        27/
          20260427-153012-a1b2c3.png
    summary/
      20260427-160000-aabbcc.png
  review_templates/
    default-daily.md
    short-daily.md
  review.db
```

路径规则：

- Go 内部使用 `filepath.Join`。
- Markdown 中写入相对路径，并使用 `/` 分隔。
- 数据库中保存相对 `<dataDir>` 的路径。
- 所有写入路径必须经过安全校验，禁止 `../` 逃逸到模块目录外。

## 6. 数据库设计

### 6.1 数据库初始化

`ReviewService` 初始化时执行：

1. 创建目录。
2. 打开 `review.db`。
3. 执行 `PRAGMA journal_mode=WAL`。
4. 执行 `PRAGMA busy_timeout=5000`。
5. 创建表。
6. 创建索引。
7. 初始化内置模板。
8. 初始化总复盘总结文章。
9. 执行一次轻量索引校验。

### 6.2 表结构

`review_articles`：

```sql
CREATE TABLE IF NOT EXISTS review_articles (
  id TEXT PRIMARY KEY,
  type TEXT NOT NULL,
  date TEXT,
  title TEXT NOT NULL,
  file_path TEXT NOT NULL UNIQUE,
  template_id TEXT,
  template_name TEXT,
  summary TEXT,
  tags TEXT NOT NULL DEFAULT '[]',
  stocks TEXT NOT NULL DEFAULT '[]',
  profit_loss REAL DEFAULT 0,
  emotion TEXT DEFAULT '',
  discipline_score INTEGER,
  image_count INTEGER DEFAULT 0,
  created_at INTEGER NOT NULL,
  updated_at INTEGER NOT NULL
);
```

`review_templates`：

```sql
CREATE TABLE IF NOT EXISTS review_templates (
  id TEXT PRIMARY KEY,
  name TEXT NOT NULL,
  description TEXT DEFAULT '',
  file_path TEXT NOT NULL UNIQUE,
  is_builtin INTEGER NOT NULL DEFAULT 0,
  is_default INTEGER NOT NULL DEFAULT 0,
  created_at INTEGER NOT NULL,
  updated_at INTEGER NOT NULL
);
```

`review_assets`：

```sql
CREATE TABLE IF NOT EXISTS review_assets (
  id TEXT PRIMARY KEY,
  article_id TEXT,
  source_type TEXT NOT NULL,
  source_url TEXT DEFAULT '',
  hash TEXT NOT NULL,
  file_path TEXT NOT NULL UNIQUE,
  mime_type TEXT NOT NULL,
  size INTEGER NOT NULL,
  created_at INTEGER NOT NULL
);
```

推荐索引：

```sql
CREATE INDEX IF NOT EXISTS idx_review_articles_type ON review_articles(type);
CREATE INDEX IF NOT EXISTS idx_review_articles_date ON review_articles(date);
CREATE INDEX IF NOT EXISTS idx_review_articles_updated_at ON review_articles(updated_at);
CREATE INDEX IF NOT EXISTS idx_review_assets_article_id ON review_assets(article_id);
CREATE INDEX IF NOT EXISTS idx_review_assets_hash ON review_assets(hash);
```

### 6.3 JSON 字段

以下字段以 JSON 字符串保存：

- `review_articles.tags`：`[]string`
- `review_articles.stocks`：`[]string`

读写时由 Go 服务转换，前端只接收数组。

### 6.4 数据库与文件一致性

保存文章顺序：

1. 规范化 Markdown 内容。
2. 解析或补齐 Front Matter。
3. 写入临时文件。
4. rename 临时文件为目标 `.md`。
5. 扫描文章图片引用。
6. 写入或更新 SQLite 索引。

如果第 5 或第 6 步失败，文章文件仍视为保存成功，但 API 应返回 `warning` 字段或日志记录，前端可提示“文章已保存，索引更新失败，可重建索引”。

## 7. Markdown 与 Front Matter 规范

### 7.1 每日复盘 Front Matter

```yaml
---
type: daily_review
date: 2026-04-27
title: 2026-04-27 每日复盘
templateId: default-daily
templateName: 标准每日复盘
summary: ""
tags: []
stocks: []
profitLoss: 0
emotion: ""
disciplineScore: null
createdAt: 2026-04-27T15:30:00+08:00
updatedAt: 2026-04-27T15:30:00+08:00
---
```

### 7.2 总复盘总结 Front Matter

```yaml
---
type: summary_review
title: 复盘总结
summary: 长期交易经验沉淀
tags: []
stocks: []
createdAt: 2026-04-27T15:30:00+08:00
updatedAt: 2026-04-27T15:30:00+08:00
---
```

### 7.3 Front Matter 解析规则

- 文件以 `---` 开头且存在第二个 `---` 时，解析中间内容。
- 解析失败时不阻断正文读取，但保存时应重写为合法 Front Matter。
- 没有 Front Matter 的外部 Markdown 文件，重建索引时按文件名推断类型和日期。
- `YYYY-MM-DD.md` 推断为每日复盘。
- `复盘总结.md` 推断为总复盘总结。

### 7.4 摘要生成规则

摘要优先级：

1. Front Matter 中的 `summary`。
2. 正文去除 Markdown 标记后的前 160 字。
3. 空字符串。

## 9. 后端服务设计

### 9.1 ReviewService 结构

```go
type ReviewService struct {
    dataDir       string
    articlesDir   string
    picturesDir   string
    templatesDir  string
    dbPath        string
    db            *sql.DB
    mu            sync.RWMutex
}
```

### 9.2 构造函数

```go
func NewReviewService(dataDir string) (*ReviewService, error)
```

职责：

- 初始化路径。
- 创建目录。
- 打开数据库。
- 执行迁移。
- 初始化默认模板。
- 初始化总复盘总结。

### 9.3 服务方法

建议实现方法：

```go
func (s *ReviewService) ListArticles(req models.ReviewListRequest) (models.ReviewListResult, error)
func (s *ReviewService) GetArticle(id string) (models.ReviewArticleDetail, error)
func (s *ReviewService) CreateDailyReview(req models.CreateDailyReviewRequest) (models.ReviewArticleDetail, error)
func (s *ReviewService) SaveArticle(req models.SaveReviewArticleRequest) (models.ReviewArticleDetail, error)
func (s *ReviewService) DeleteArticle(id string) error

func (s *ReviewService) ListTemplates() ([]models.ReviewTemplate, error)

func (s *ReviewService) SavePastedImage(req models.SaveReviewImageRequest) (models.SaveReviewImageResult, error)

func (s *ReviewService) Compare(req models.CompareReviewRequest) (models.CompareReviewResult, error)
func (s *ReviewService) RebuildIndex() error
func (s *ReviewService) Close() error
```

### 9.4 锁策略

- 读列表、读详情、读模板使用 `RLock`。
- 创建、保存、删除、图片写入、重建索引使用 `Lock`。
- 数据库操作和文件操作需要在同一个写锁内完成，避免并发保存同一文章导致内容覆盖。

### 9.5 错误处理

后端服务返回错误时应包含明确中文信息，便于前端直接展示。

错误示例：

- `复盘日期格式错误，请使用 YYYY-MM-DD`
- `该日期复盘已存在`
- `文章不存在`
- `内置模板不可删除`
- `图片格式不支持`
- `图片超过大小限制`
- `网络图片下载超时`
- `图片链接不是有效的 http/https 地址`

## 10. Wails API 设计

### 10.1 App 初始化

在 `App` 结构体中新增字段：

```go
reviewService *services.ReviewService
```

在 `NewApp()` 中初始化：

```go
reviewService, err := services.NewReviewService(dataDir)
if err != nil {
    log.Warn("Review service init failed: %v", err)
}
```

如果初始化失败：

- 不应阻断主应用启动。
- API 调用时返回明确错误。

### 10.2 App 关闭

在 `shutdown()` 中关闭数据库：

```go
if a.reviewService != nil {
    _ = a.reviewService.Close()
}
```

### 10.3 API 方法

建议在 `app.go` 末尾新增复盘相关方法，命名保持 Wails 风格。

```go
func (a *App) GetReviewArticles(req models.ReviewListRequest) models.ReviewListResult
func (a *App) GetReviewArticle(id string) models.ReviewArticleDetail
func (a *App) CreateDailyReview(req models.CreateDailyReviewRequest) models.ReviewArticleDetail
func (a *App) SaveReviewArticle(req models.SaveReviewArticleRequest) models.ReviewArticleDetail
func (a *App) DeleteReviewArticle(id string) string
func (a *App) GetReviewSummaryArticle() models.ReviewArticleDetail
```

## 11. 后端关键流程

### 11.1 初始化流程

```text
NewReviewService
  -> resolve paths
  -> mkdir articles/pictures/review_templates
  -> open review.db
  -> migrate tables
  -> ensure default templates
  -> ensure summary article
  -> return service
```

### 11.2 创建每日复盘流程

```text
CreateDailyReview(req)
  -> validate date
  -> check existing article by date
  -> load template by req.TemplateID or default
  -> render template variables
  -> build front matter
  -> write articles/YYYY-MM-DD.md
  -> parse metadata
  -> upsert review_articles
  -> return detail
```

变量渲染：

- `{{date}}` 替换为 `YYYY-MM-DD`。
- `{{weekday}}` 替换为 `星期一` 至 `星期日`。
- `{{title}}` 替换为文章标题。

### 11.3 保存文章流程

```text
SaveArticle(req)
  -> load existing index
  -> normalize front matter
  -> update updatedAt/title
  -> atomic write file
  -> scan image references
  -> upsert article index
  -> return latest detail
```

原子写入实现：

```text
target: articles/2026-04-27.md
tmp:    articles/.2026-04-27.md.tmp
write tmp
fsync if necessary
rename tmp -> target
```

### 11.4 删除文章流程

```text
DeleteArticle(id)
  -> load article
  -> reject summary_review
  -> remove markdown file
  -> delete review_articles row
  -> set review_assets.article_id = NULL for this article
```

### 11.5 粘贴图片流程

```text
Frontend paste image
  -> read file/blob
  -> convert to base64
  -> call SaveReviewPastedImage

Backend SaveReviewPastedImage
  -> validate article/date
  -> decode base64
  -> validate mime and size
  -> sha256 hash
  -> generate file name
  -> ensure target dir
  -> write image
  -> insert review_assets
  -> return Markdown text

Frontend
  -> insert Markdown text at cursor
  -> mark dirty
```

### 11.6 网络图片下载流程

```text
Frontend paste URL or markdown image URL
  -> detect image URL
  -> call DownloadReviewImage

Backend DownloadReviewImage
  -> validate URL scheme
  -> create http client with timeout
  -> GET URL
  -> check status code
  -> read with max bytes limit
  -> validate content type
  -> hash content
  -> write image
  -> insert review_assets
  -> return Markdown path

Frontend
  -> replace original URL with local path
```

### 11.7 重建索引流程

```text
RebuildIndex
  -> scan articles/*.md
  -> parse each file front matter
  -> infer missing metadata
  -> scan image refs
  -> clear review_articles
  -> upsert parsed articles
  -> keep review_assets rows
  -> relink assets by file_path
```

## 16. 图片路径与预览方案

### 16.1 Markdown 中的图片路径

文章位于：

```text
articles/2026-04-27.md
```

图片位于：

```text
pictures/2026/04/27/20260427-153012-a1b2c3.png
```

Markdown 写入：

```markdown
![截图](../pictures/2026/04/27/20260427-153012-a1b2c3.png)
```

### 16.2 前端预览路径问题

浏览器无法直接用相对 URL 读取应用数据目录。因此实现时必须选择一种安全访问方案。

推荐首期方案：

- 后端提供 `ResolveReviewImageRefs(content string)` 或 `GetReviewAssetBase64(filePath string)`。
- 前端预览前提取图片路径并批量请求 base64。
- 将 Markdown 中的本地图片路径替换为 data URL 后交给 `NodeRenderer`。

更优后续方案：

- 用 Wails 支持的安全本地文件服务或自定义协议映射 pictures 目录。
- 前端直接使用安全 URL 加载图片，降低 base64 内存开销。

## 18. 安全与边界处理

### 18.1 路径安全

所有传入后端的路径都必须执行：

```text
clean path
join base dir
abs target
abs base
check target starts with base
```

禁止：

- `../`
- 绝对路径。
- Windows 盘符路径。
- UNC 路径。
- 空字节。

### 18.2 文件名安全

文章文件名：

- 每日复盘固定为 `YYYY-MM-DD.md`。
- 总复盘总结固定为 `复盘总结.md`。

图片文件名：

- 不使用用户传入原始文件名作为最终文件名。
- 格式：`YYYYMMDD-HHMMSS-{hash6}.{ext}`。
- 扩展名由 MIME 类型决定。

模板文件名：

- 使用模板 ID。
- 模板 ID 用 UUID 或安全 slug。

### 18.3 网络下载安全

规则：

- 只允许 `http` 和 `https`。
- 禁止空 host。
- 禁止 `file://`、`ftp://`、`data:`。
- 设置超时。
- 限制最大响应体。
- 校验 Content-Type。
- 读取完成后再次用 `http.DetectContentType` 校验。

### 18.4 并发保存

同一文章多次保存：

- 后端用写锁串行化。
- 前端自动保存应防抖，并避免保存中再次发起保存。
- 如果用户手动保存时已有自动保存进行中，手动保存应等待或取消前一次请求。

- 文章列表能显示总复盘总结和每日复盘。
- 能新建指定日期复盘。
- 能选择模板。
- 能编辑、保存、预览 Markdown。
- 能粘贴图片。
- 能下载网络图片。
- 能管理模板。
- 能进行多日对比。

数据验收：

- Markdown 文件在 `articles` 目录。
- 图片文件在 `pictures` 目录。
- 模板文件在 `review_templates` 目录。
- 文章中的图片链接为本地相对路径。
- 删除 `review.db` 后可通过重建索引恢复文章列表。

## 24. 风险与应对

### 24.1 本地图片预览风险

风险：

- 前端无法直接读取应用数据目录图片。

应对：

- 首期使用后端 base64 API。
- 后续优化为安全本地文件协议。

### 24.2 Markdown Front Matter 解析风险

风险：

- 用户手动改坏 Front Matter。

应对：

- 读取时容错。
- 保存时重写合法 Front Matter。
- 正文优先保留。

### 24.3 SQLite 与文件不一致风险

风险：

- 文件保存成功但数据库更新失败。

应对：

- Markdown 文件作为真源。
- 提供重建索引。
- 保存时返回 warning。

### 24.4 大图片导致卡顿

风险：

- base64 预览大图内存占用较高。

应对：

- 限制单图大小。
- 首期默认 10 MB。
- 后续使用本地文件 URL。

### 24.5 网络图片下载不可控

风险：

- 网络慢、响应过大、类型伪造。

应对：

- 超时。
- 限制响应大小。
- 校验 MIME。
- 失败时保留原始 URL，并提示用户。

## 25. 推荐首版交付顺序

建议严格按以下顺序开发，降低集成风险：

1. 后端模型和 ReviewService 初始化。
2. 默认模板和总复盘总结。
3. 文章 CRUD 和 Wails API。
4. 前端入口和 ReviewDialog。
5. 文章列表、新建、打开。
6. Markdown 编辑和保存。
7. Markdown 预览。
8. 粘贴图片保存。
9. 网络图片下载。
10. 模板管理。
11. 多日对比。
12. 索引重建入口。
13. 全量构建和回归测试。
