# Epic E3 · 前端复盘工作台与 Markdown 闭环

## 1. Epic 概览

- **Epic ID：** E3
- **目标：** 在 React/Wails 前端中新增每日复盘入口、工作台、文章列表、Markdown 编辑、预览、保存和总复盘总结编辑能力。
- **业务价值：** 用户可以在桌面端完成从打开复盘模块到创建、编辑、保存、预览文章的首期闭环。
- **范围内：** `reviewService.ts`、主界面入口、`ReviewDialog`、`ReviewArticleList`、`ReviewEditor`、`ReviewPreview`、未保存提示、自动保存、快捷键保存。
- **范围外：** 图片粘贴和网络图片下载、多日对比具体展示、完整富文本编辑器。
- **上游依赖：** E2 的文章 Wails API；Wails 绑定已生成或可生成。
- **风险/源文档缺口：** Markdown 图片预览需要 E4 的安全本地图片访问能力；本 Epic 先完成文本预览和缺图占位。

---

## 2. Story 列表

### Story E3-S1 · 复盘入口和服务封装

**用户故事：** 作为用户，我希望从主界面快速打开每日复盘工作台，以便不离开股票分析流程就能记录复盘。

**验收摘要：**
- AC1: 顶部工具区存在“每日复盘”入口。
- AC2: 前端有统一服务封装调用 Wails API。

#### Issue E3-S1-I1 · 新增前端复盘服务封装和类型

- **目标：** 创建 `frontend/src/services/reviewService.ts`，封装复盘 Wails API 和 TypeScript 类型。
- **范围：** 定义 `ReviewArticle`、`ReviewArticleDetail`、`ReviewTemplate`、`ReviewListRequest`、`CompareReviewResult`、图片请求响应等类型；封装文章列表、详情、创建、保存、删除、总结、模板、图片、对比、重建索引 API。
- **依赖：** E2-S3-I2。
- **备注/风险：** 新增 Wails API 后必须重新生成绑定，否则导入路径会失败。

##### 验收标准
- AC1: 类型字段与 Go JSON 字段一致。
- AC2: 删除和重建类 API 对非 `"success"` 返回按失败处理。
- AC3: 详情类响应可透传 `warning` 或 `error`。

### Story E3-S3 · Markdown 编辑、预览和保存体验

**用户故事：** 作为用户，我希望能编辑、预览并可靠保存复盘文章，以便完成每日记录闭环。

**验收摘要：**
- AC1: 支持编辑、预览、左右分栏三种模式。
