# Epic E4 · 图片归档、网络下载与预览

## 1. Epic 概览

- **Epic ID：** E4
- **目标：** 实现剪贴板/拖拽图片保存、网络图片下载、本地 Markdown 链接替换、图片资产索引和安全预览。
- **业务价值：** 让复盘截图和网络图片自动归档到本地 `pictures` 目录，避免图片散落和文章链接失效。
- **范围内：** 后端图片保存、网络下载、MIME/大小/路径安全校验、资产索引、图片 base64 安全读取、前端粘贴拖拽和 URL 替换。
- **范围外：** 图片清理器、孤立图片回收站、高级本地文件协议、图片导出。
- **上游依赖：** E1 路径安全基础、E2 文章索引、E3 编辑器事件入口。
- **风险/源文档缺口：** Wails 前端无法直接读取应用数据目录图片，首期采用后端 base64 API，后续再优化为安全本地协议。

---

## 2. Story 列表

### Story E4-S1 · 后端图片保存和下载

**用户故事：** 作为用户，我希望粘贴截图或网络图片时系统自动保存到本地，以便复盘文章长期可读。

**验收摘要：**
- AC1: 粘贴图片保存到 `pictures/YYYY/MM/DD/` 或 `pictures/summary/`。
- AC2: 网络图片只允许 `http/https`，并校验大小、类型和超时。

#### Issue E4-S1-I1 · 实现粘贴图片保存接口

- **目标：** 实现 `SavePastedImage(req)`，保存前端传入的 base64 图片。
- **范围：** 校验 article/date；解析 base64；支持 png、jpeg、webp、gif；默认最大 10 MB；sha256 hash；按日期生成文件名；写入 `review_assets`；返回 Markdown 文本。
- **依赖：** E1-S3-I2、E2-S1-I2。
- **备注/风险：** 不使用用户原始文件名作为最终文件名。

##### 验收标准
- AC1: 合法图片可保存并返回 `MarkdownText`。
- AC2: 返回 `FilePath` 相对 `<dataDir>`，`MarkdownPath` 相对 `articles` 目录。
- AC3: 相同内容 hash 可复用或至少避免路径冲突。

##### 功能测试
| 编号 | 用例名称 | 前置条件 | 输入/操作 | 预期结果 |
|------|----------|----------|-----------|----------|
| FT-1 | 保存 PNG 截图 | 已有每日复盘 | 上传小于 10 MB 的 png base64 | 文件写入日期目录并返回 Markdown 图片语法 |
| FT-2 | 总结文章图片目录 | 当前文章为复盘总结 | 上传图片 | 图片保存到 `pictures/summary/` |

##### 边界测试
| 编号 | 用例名称 | 前置条件 | 输入/操作 | 预期结果 |
|------|----------|----------|-----------|----------|
| BT-1 | 超大图片 | 图片大于 10 MB | 调用保存 | 返回图片超过大小限制 |
| BT-2 | 非图片 MIME | 输入 `text/plain` | 调用保存 | 返回图片格式不支持 |

#### Issue E4-S1-I2 · 实现网络图片下载接口

- **目标：** 实现 `DownloadImage(req)`，下载网络图片并归档为本地图片。
- **范围：** 校验 URL scheme 和 host；15 秒超时；限制最大响应体；校验 HTTP 状态、Content-Type 和 `http.DetectContentType`；保存文件和资产索引；相同 URL 或 hash 复用。
- **依赖：** E4-S1-I1。
- **备注/风险：** 下载失败时前端应保留原始 URL 并提示，后端不应写入半文件。

##### 验收标准
- AC1: 合法 http/https 图片可下载并返回本地 Markdown 路径。
- AC2: 非图片、超时、超大、错误状态码都返回明确错误。
- AC3: 文件名由日期、时间和 hash 生成。

##### 功能测试
| 编号 | 用例名称 | 前置条件 | 输入/操作 | 预期结果 |
|------|----------|----------|-----------|----------|
| FT-1 | 下载 HTTPS 图片 | 网络可访问测试图片 | 调用下载接口 | 图片保存并返回 MarkdownText |
| FT-2 | 重复 URL 复用 | 同一 URL 已下载 | 再次下载 | 返回已有或相同 hash 资产，不重复无意义写入 |

##### 边界测试
| 编号 | 用例名称 | 前置条件 | 输入/操作 | 预期结果 |
|------|----------|----------|-----------|----------|
| BT-1 | 禁止 file 协议 | 输入 `file:///a.png` | 调用下载 | 返回无效协议错误 |
| BT-2 | Content-Type 伪造 | 响应声明图片但内容不是图片 | 调用下载 | DetectContentType 校验失败并拒绝 |

#### Issue E4-S1-I3 · 暴露图片和安全预览 Wails API

- **目标：** 在 `app.go` 暴露 `SaveReviewPastedImage`、`DownloadReviewImage` 和安全图片读取接口。
- **范围：** 新增 `GetReviewAssetBase64(filePath string)` 或等效 API；只允许读取 `pictures` 下相对路径；返回 data URL；处理不存在图片。
- **依赖：** E4-S1-I2。
- **备注/风险：** 该 API 不能成为任意本地文件读取入口。

##### 验收标准
- AC1: 前端可调用图片保存和下载 API。
- AC2: 本地图片可转换为 `data:image/...;base64,...` 供预览。
- AC3: 路径穿越和非 pictures 路径被拒绝。

##### 功能测试
| 编号 | 用例名称 | 前置条件 | 输入/操作 | 预期结果 |
|------|----------|----------|-----------|----------|
| FT-1 | 读取已保存图片 | 图片已保存 | 调用 `GetReviewAssetBase64` | 返回 data URL |
| FT-2 | Wails 图片保存 | App 初始化成功 | 调用 `SaveReviewPastedImage` | 返回保存结果 |

##### 边界测试
| 编号 | 用例名称 | 前置条件 | 输入/操作 | 预期结果 |
|------|----------|----------|-----------|----------|
| BT-1 | 路径穿越读取 | 输入 `../articles/a.md` | 调用 base64 API | 返回路径非法错误 |

### Story E4-S2 · 前端图片交互

**用户故事：** 作为用户，我希望在编辑器中粘贴截图、拖拽图片或粘贴网络图片链接后自动插入本地 Markdown 图片链接，以便减少整理成本。

**验收摘要：**
- AC1: 图片上传成功后在光标位置插入 Markdown 图片语法。
- AC2: 网络图片 URL 或 Markdown 图片 URL 可替换为本地路径。

#### Issue E4-S2-I1 · 实现编辑器粘贴和拖拽图片上传

- **目标：** 在 `ReviewEditor` 中处理剪贴板和拖拽图片。
- **范围：** 读取 clipboard/drop 中的 `image/png`、`image/jpeg`、`image/webp`、`image/gif`；转 base64；调用保存 API；按 textarea 光标位置插入 `MarkdownText`；标记 dirty。
- **依赖：** E4-S1-I3、E3-S3-I1。
- **备注/风险：** 上传期间需要显示状态，失败不应清空用户原有文本。

##### 验收标准
- AC1: 粘贴截图后 Markdown 图片语法插入光标位置。
- AC2: 拖拽图片到编辑器可完成相同上传。
- AC3: 上传失败保留编辑器内容并展示错误。

##### 功能测试
| 编号 | 用例名称 | 前置条件 | 输入/操作 | 预期结果 |
|------|----------|----------|-----------|----------|
| FT-1 | 粘贴截图 | 打开每日复盘编辑器 | 从剪贴板粘贴 png | 调用保存接口并插入 MarkdownText |
| FT-2 | 拖拽图片 | 打开编辑器 | 拖入 jpg 文件 | 上传成功并插入图片链接 |

##### 边界测试
| 编号 | 用例名称 | 前置条件 | 输入/操作 | 预期结果 |
|------|----------|----------|-----------|----------|
| BT-1 | 粘贴非图片文本 | 打开编辑器 | 粘贴普通文本 | 保持浏览器默认文本粘贴 |
| BT-2 | 上传失败 | 后端返回超大图片错误 | 粘贴图片 | 显示错误，正文不改变或可恢复 |

#### Issue E4-S2-I2 · 实现网络图片识别、替换和预览加载

- **目标：** 在前端识别网络图片 URL 和 Markdown 图片语法，下载后替换为本地路径，并让预览可显示本地图片。
- **范围：** 识别 `![alt](https://...)` 和直接图片 URL；调用下载 API；替换选区或匹配 URL；预览前把 `../pictures/...` 映射为 base64 data URL；缺图显示占位。
- **依赖：** E4-S2-I1、E3-S3-I2。
- **备注/风险：** URL 下载失败时保留原始文本并提示。

##### 验收标准
- AC1: 直接粘贴网络图片 URL 可插入本地 Markdown 图片链接。
- AC2: 粘贴 Markdown 网络图片语法可替换括号内 URL。
- AC3: 重新打开文章后本地图片可在预览中展示。

##### 功能测试
| 编号 | 用例名称 | 前置条件 | 输入/操作 | 预期结果 |
|------|----------|----------|-----------|----------|
| FT-1 | 直接 URL 下载 | 打开编辑器 | 粘贴 `https://.../a.png` | 下载成功并插入本地图片 Markdown |
| FT-2 | Markdown 图片替换 | 打开编辑器 | 粘贴 `![图](https://.../a.jpg)` | URL 被替换为 `../pictures/...` |

##### 边界测试
| 编号 | 用例名称 | 前置条件 | 输入/操作 | 预期结果 |
|------|----------|----------|-----------|----------|
| BT-1 | 网络下载失败 | URL 超时或 404 | 粘贴 URL | 原始 URL 保留并提示失败 |
| BT-2 | 图片 base64 读取失败 | 文章引用不存在图片 | 切换预览 | 显示缺图占位，正文继续渲染 |
