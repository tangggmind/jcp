# Epic E1 · 复盘数据与服务基础

## 1. Epic 概览

- **Epic ID：** E1
- **目标：** 建立每日复盘模块的后端模型、目录、数据库、默认内容和基础文件处理能力。
- **业务价值：** 让复盘内容具备可靠的本地 Markdown 真源、SQLite 索引和可恢复的数据基础。
- **范围内：** Go 模型、请求响应结构、ReviewService 初始化、目录创建、SQLite 表和索引、默认模板、总复盘总结、Front Matter 与路径安全基础工具。
- **范围外：** 前端工作台、文章 CRUD 交互、图片上传下载、多日对比、AI 或行情自动填充。
- **上游依赖：** `paths.GetDataDir()` 可用；项目已具备 Wails v2、Go 1.24 和 SQLite 依赖。
- **风险/源文档缺口：** PRD 的阶段规划把图片和对比放在后续阶段，功能开发说明书把它们列入首期目标；本拆分按依赖顺序排期，先完成本 Epic 后再进入图片与对比能力。

## 2. Story 列表

### Story E1-S1 · 数据模型和数据库基础

**用户故事：** 作为开发者，我希望先定义复盘模块的数据模型和数据库结构，以便后续功能具备稳定的数据契约。

**验收摘要：**
- AC1: 复盘文章、模板、图片、列表、保存、对比等模型字段与开发说明书保持一致。
- AC2: `review.db` 可自动创建表和推荐索引，并使用纯 Go SQLite 驱动。

#### Issue E1-S1-I1 · 定义复盘模型与请求响应结构

- **目标：** 新增 `internal/models/review.go`，定义复盘模块常量、文章、模板、图片、列表、保存、对比等模型。
- **范围：** 包含 `ReviewArticle`、`ReviewArticleDetail`、`ReviewListRequest`、`ReviewListResult`、`CreateDailyReviewRequest`、`SaveReviewArticleRequest`、`ReviewTemplate`、`SaveReviewTemplateRequest`、`SaveReviewImageRequest`、`DownloadReviewImageRequest`、`SaveReviewImageResult`、`CompareReviewRequest`、`CompareReviewResult` 等结构。
- **依赖：** 无。
- **备注/风险：** 字段 JSON 名称必须与功能开发说明书一致，避免 Wails 绑定后前端类型不匹配。

##### 验收标准
- AC1: 所有源文档明确列出的模型字段均存在，字段类型符合 Go 与 JSON 传输需求。
- AC2: `daily_review`、`summary_review`、`paste`、`download`、`local` 等常量可复用。
- AC3: 可选字段如 `disciplineScore` 使用指针或等效方式表达空值。

#### Issue E1-S1-I2 · 初始化 ReviewService 目录和 SQLite 迁移

### Story E1-S3 · 文件解析和安全基础工具

**用户故事：** 作为开发者，我希望复盘模块具备统一的 Front Matter、路径安全和原子写入工具，以便所有文件操作可靠且可恢复。

**验收摘要：**
- AC1: 文章保存和索引重建可容错解析 Front Matter。
- AC2: 所有写入路径均限制在复盘数据目录内，禁止路径穿越。

#### Issue E1-S3-I1 · 实现 Front Matter、摘要和图片引用解析工具

- **目标：** 实现 Markdown Front Matter 解析、补齐、摘要生成和图片引用扫描工具。
- **范围：** 支持 `---` 包裹的 YAML 风格元数据；解析失败不阻断正文读取；摘要优先 Front Matter `summary`，其次正文前 160 字；扫描本地 Markdown 图片引用用于 `image_count`。
- **依赖：** E1-S1-I1。
- **备注/风险：** 不要求引入完整 YAML 高级能力，但必须覆盖源文档字段读写。

##### 验收标准
- AC1: 合法 Front Matter 可解析为文章元数据。
- AC2: 缺失或损坏 Front Matter 时可保留正文并在保存时重写合法 Front Matter。
- AC3: Markdown 标记可被清理后生成摘要。

##### 功能测试
| 编号 | 用例名称 | 前置条件 | 输入/操作 | 预期结果 |
|------|----------|----------|-----------|----------|
| FT-1 | 解析每日复盘 Front Matter | 输入标准示例 | 调用解析工具 | 返回类型、日期、标题、标签、股票、盈亏等字段 |
| FT-2 | 摘要生成 | Front Matter summary 为空 | 输入带 Markdown 标题和正文 | 返回去除标记后的前 160 字以内摘要 |

##### 边界测试
| 编号 | 用例名称 | 前置条件 | 输入/操作 | 预期结果 |
|------|----------|----------|-----------|----------|
| BT-1 | Front Matter 只有开始分隔符 | 输入异常 Markdown | 调用解析工具 | 不 panic，正文可读取并标记需要重写 |
| BT-2 | 图片引用混合远程和本地 | 输入含 `https://` 与 `../pictures/` 图片 | 扫描图片引用 | 仅本地图片计入本地图片数量 |

#### Issue E1-S3-I2 · 实现路径安全校验和原子写入工具

- **目标：** 为文章、模板、图片读写提供统一的安全路径解析和临时文件 rename 写入能力。
- **范围：** 禁止 `../`、绝对路径、Windows 盘符、UNC、空字节；写文件使用临时文件加 rename；Markdown 路径使用 `/` 分隔。
- **依赖：** E1-S1-I2。
- **备注/风险：** Windows 下路径大小写和分隔符处理必须谨慎，不能只做字符串包含判断。

##### 验收标准
- AC1: 任意用户输入路径都不能逃逸 `articles`、`pictures`、`review_templates` 允许目录。
- AC2: 文件写入异常时不破坏已有正式文件。
- AC3: 返回给 Markdown 的相对路径统一使用 `/`。

| 编号 | 用例名称 | 前置条件 | 输入/操作 | 预期结果 |
|------|----------|----------|-----------|----------|
| FT-1 | 合法文章路径解析 | dataDir 已初始化 | 解析 `articles/2026-04-27.md` | 返回 dataDir 内绝对路径 |
| FT-2 | 原子写入成功 | 已有文章文件 | 调用原子写入新内容 | 正式文件内容完整替换 |

##### 边界测试
| 编号 | 用例名称 | 前置条件 | 输入/操作 | 预期结果 |
|------|----------|----------|-----------|----------|
| BT-1 | 路径穿越拒绝 | dataDir 已初始化 | 输入 `../secret.txt` | 返回路径非法错误 |
| BT-2 | 写入中断保护 | 已有文章文件 | 模拟临时文件写入失败 | 旧正式文件仍可读取 |
