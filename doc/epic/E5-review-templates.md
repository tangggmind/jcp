# Epic E5 · 复盘模板管理

## 1. Epic 概览

- **Epic ID：** E5
- **目标：** 实现复盘模板的后端管理、前端管理界面、默认模板规则和新建文章模板变量渲染。
- **业务价值：** 用户可以用模板降低每日输入成本，并保持复盘结构长期一致。
- **范围内：** 模板 CRUD、内置模板保护、默认模板设置、模板管理 UI、模板复制、创建文章时选择自定义模板和变量渲染。
- **范围外：** 富文本模板编辑器、模板市场、AI 自动生成模板。
- **上游依赖：** E1 默认模板初始化；E2 创建文章；E3 前端工作台。
- **风险/源文档缺口：** PRD 提到“从当前文章另存为模板”，功能开发说明书更强调模板管理和内置模板复制；首期按模板 CRUD 与复制能力实现，文章另存模板可作为后续扩展。

---

## 2. Story 列表

### Story E5-S1 · 模板后端管理

**用户故事：** 作为用户，我希望可以新增、编辑、删除和设置默认模板，以便按自己的复盘习惯创建文章。

**验收摘要：**
- AC1: 模板正文保存为 Markdown 文件，元数据写入 SQLite。
- AC2: 内置模板不可删除，默认模板必须始终存在。

#### Issue E5-S1-I1 · 实现模板 CRUD 和默认模板规则

- **目标：** 实现 `ListTemplates`、`SaveTemplate`、`DeleteTemplate`。
- **范围：** 新建模板；编辑模板；删除自定义模板；设置默认模板；禁止删除内置模板；删除默认模板前必须先切换其他默认模板；模板文件路径为 `review_templates/{templateId}.md`。
- **依赖：** E1-S2-I1、E1-S3-I2。
- **备注/风险：** 模板 ID 需使用 UUID 或安全 slug，避免路径注入。

##### 验收标准
- AC1: 模板列表返回正文和元数据。
- AC2: 设置某模板为默认时，其他模板默认标记清除。
- AC3: 内置模板删除失败，自定义模板可删除。

##### 功能测试
| 编号 | 用例名称 | 前置条件 | 输入/操作 | 预期结果 |
|------|----------|----------|-----------|----------|
| FT-1 | 新建自定义模板 | 服务已初始化 | 保存无 ID 模板 | 生成安全 ID、文件和数据库记录 |
| FT-2 | 设置默认模板 | 已有两个模板 | 保存其中一个 `IsDefault=true` | 该模板为默认，其他模板取消默认 |

##### 边界测试
| 编号 | 用例名称 | 前置条件 | 输入/操作 | 预期结果 |
|------|----------|----------|-----------|----------|
| BT-1 | 删除内置模板 | 默认模板存在 | 删除 `default-daily` | 返回“内置模板不可删除” |
| BT-2 | 模板名称为空 | 新建模板 | `Name=""` | 返回模板名称必填错误 |

#### Issue E5-S1-I2 · 暴露模板 Wails API

- **目标：** 在 `app.go` 中暴露模板相关 API。
- **范围：** `GetReviewTemplates`、`SaveReviewTemplate`、`DeleteReviewTemplate`；错误返回约定与前端服务封装一致。
- **依赖：** E5-S1-I1。
- **备注/风险：** 删除类接口返回非 `"success"` 时前端应视为失败。

##### 验收标准
- AC1: Wails 绑定包含模板 API。
- AC2: 模板保存错误可在前端展示。
- AC3: ReviewService 未初始化时模板 API 不 panic。

##### 功能测试
| 编号 | 用例名称 | 前置条件 | 输入/操作 | 预期结果 |
|------|----------|----------|-----------|----------|
| FT-1 | 获取模板列表 API | App 初始化成功 | 调用 `GetReviewTemplates` | 返回模板数组 |
| FT-2 | 保存模板 API | App 初始化成功 | 调用 `SaveReviewTemplate` | 返回保存后的模板 |

##### 边界测试
| 编号 | 用例名称 | 前置条件 | 输入/操作 | 预期结果 |
|------|----------|----------|-----------|----------|
| BT-1 | 删除不存在模板 | ID 不存在 | 调用删除 API | 返回明确错误文本 |
| BT-2 | 服务为空 | `reviewService=nil` | 调用模板 API | 返回错误，不 panic |

### Story E5-S2 · 模板前端管理和创建集成

**用户故事：** 作为用户，我希望在复盘工作台中管理模板，并在新建复盘时选择模板，以便快速生成适合自己的文章结构。

**验收摘要：**
- AC1: 模板管理弹窗支持新建、编辑、删除、设置默认和复制。
- AC2: 新建每日复盘时可使用自定义模板。

#### Issue E5-S2-I1 · 实现 ReviewTemplateDialog 模板管理界面

- **目标：** 创建 `ReviewTemplateDialog.tsx`，提供模板管理 UI。
- **范围：** 模板列表；新建模板；编辑模板；删除模板；设置默认；内置和默认标识；内置模板不可删除但可复制为自定义模板。
- **依赖：** E5-S1-I2、E3-S2-I1。
- **备注/风险：** 模板正文可用 textarea 编辑，不引入额外编辑器。

##### 验收标准
- AC1: 用户可在复盘工作台打开模板管理弹窗。
- AC2: 保存模板后刷新模板列表。
- AC3: 内置模板删除按钮禁用或操作后提示不可删除。

##### 功能测试
| 编号 | 用例名称 | 前置条件 | 输入/操作 | 预期结果 |
|------|----------|----------|-----------|----------|
| FT-1 | 新建模板 UI | 模板弹窗打开 | 输入名称和内容保存 | 列表出现新模板 |
| FT-2 | 设置默认 UI | 列表有自定义模板 | 点击设为默认 | 默认标识切换到该模板 |

##### 边界测试
| 编号 | 用例名称 | 前置条件 | 输入/操作 | 预期结果 |
|------|----------|----------|-----------|----------|
| BT-1 | 删除内置模板 UI | 内置模板显示 | 点击删除 | 删除不可用或显示错误 |
| BT-2 | 未保存模板切换 | 正在编辑模板且 dirty | 切换模板 | 出现确认或保留草稿，避免丢失 |

#### Issue E5-S2-I2 · 集成模板选择、变量渲染和模板复制

- **目标：** 将模板管理能力与新建每日复盘流程打通。
- **范围：** 新建弹窗展示模板描述和默认模板；创建时传入 TemplateID；模板复制生成自定义模板；创建后的文章记录 `templateId` 和 `templateName`。
- **依赖：** E5-S2-I1、E3-S2-I3、E2-S1-I1。
- **备注/风险：** 变量渲染主要在后端创建文章完成，前端只负责选择模板。

##### 验收标准
- AC1: 自定义模板可用于创建每日复盘。
- AC2: 创建后的 Front Matter 包含正确模板 ID 和模板名称。
- AC3: 复制内置模板后可编辑为自定义模板。

|------|----------|----------|-----------|----------|
| FT-1 | 使用自定义模板创建 | 已有自定义模板 | 新建复盘选择该模板 | 文章正文来自自定义模板 |
| FT-2 | 复制内置模板 | 内置模板存在 | 点击复制 | 生成可编辑自定义模板 |

##### 边界测试
| 编号 | 用例名称 | 前置条件 | 输入/操作 | 预期结果 |
|------|----------|----------|-----------|----------|

| BT-2 | 模板变量未知 | 模板含 `{{unknown}}` | 创建文章 | 未知变量保留原样，不影响已支持变量渲染 |
