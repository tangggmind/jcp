# Epic E2 · 文章管理与索引闭环

## 1. Epic 概览

- **Epic ID：** E2
- **目标：** 实现每日复盘和总复盘总结的创建、读取、保存、删除、列表、搜索、筛选、索引同步与 Wails API 暴露。
- **业务价值：** 让用户可以把交易日复盘稳定保存为 Markdown 文件，并通过索引快速检索和恢复。
- **范围内：** 文章 CRUD、Front Matter 同步、SQLite 索引、列表搜索筛选、总复盘置顶、删除规则、重建索引、文章相关 Wails API。
- **范围外：** 前端 UI、图片上传下载、模板管理 UI、多日对比 UI。
- **上游依赖：** E1 完成。
- **风险/源文档缺口：** 日期使用自然日还是交易日仍为待确认问题；首期按功能开发说明书采用自然日 `YYYY-MM-DD` 并允许任意日期。

---

## 2. Story 列表

### Story E2-S1 · 文章创建和读取

**用户故事：** 作为交易复盘用户，我希望按日期创建和打开复盘文章，以便记录当天操作和结论。

**验收摘要：**
- AC1: 每个自然日期默认只允许一篇每日复盘。
- AC2: 总复盘总结可读取且与每日复盘区分。

#### Issue E2-S1-I1 · 创建指定日期每日复盘

- **目标：** 实现 `CreateDailyReview(req)`，根据模板和日期生成每日复盘 Markdown 文件与索引。
- **范围：** 校验日期；检查重复日期；加载指定模板或默认模板；渲染 `{{date}}`、`{{weekday}}`、`{{title}}`；写入 Front Matter；保存到 `articles/YYYY-MM-DD.md`。
- **依赖：** E1-S2-I1、E1-S3-I2。
- **备注/风险：** 若指定模板不存在，功能说明要求提示并使用默认模板；后端需返回可展示的 warning 或明确行为。

##### 验收标准
- AC1: 合法日期可创建文章并返回 `ReviewArticleDetail`。
- AC2: 重复日期创建失败并返回“该日期复盘已存在”类错误。
- AC3: 创建内容包含合法 Front Matter 和模板渲染后的正文。

| FT-1 | 创建今日复盘 | 服务已初始化 | `Date=2026-04-27, TemplateID=default-daily` | 生成 `articles/2026-04-27.md` 并返回详情 |
| FT-2 | 模板变量渲染 | 默认模板存在 | 创建周一日期复盘 | 正文中 `{{date}}`、`{{weekday}}`、`{{title}}` 被替换 |

##### 边界测试
| 编号 | 用例名称 | 前置条件 | 输入/操作 | 预期结果 |
|------|----------|----------|-----------|----------|
| BT-1 | 日期格式错误 | 服务已初始化 | `Date=20260427` | 返回日期格式错误 |
| BT-2 | 日期已存在 | 该日期文件已存在 | 再次创建同日期 | 返回重复日期错误，不覆盖原文件 |

#### Issue E2-S1-I2 · 获取文章详情、列表搜索和筛选

- **目标：** 实现 `GetArticle(id)` 和 `ListArticles(req)`。
- **范围：** 按 ID 读取详情；列表按日期倒序；总复盘总结置顶；支持 query、type、startDate、endDate、tags、stocks、分页；摘要来源符合规则。
- **依赖：** E2-S1-I1、E1-S3-I1。
- **备注/风险：** SQLite 中 `tags`、`stocks` 为 JSON 字符串，搜索时需兼顾标题、摘要、正文或索引字段。

##### 验收标准
- AC1: 列表包含文章元数据，不返回完整正文。
- AC2: 总复盘总结置顶，每日复盘按日期倒序。
- AC3: 搜索和日期范围筛选结果正确。

| FT-1 | 列表排序 | 已有总结和两篇每日复盘 | 调用 `ListArticles` | 总结在前，每日复盘按日期倒序 |
| FT-2 | 关键词搜索 | 文章标题或正文含“纪律问题” | `Query=纪律问题` | 返回匹配文章 |

##### 边界测试

|------|----------|----------|-----------|----------|

| BT-2 | 空结果 | 无匹配文章 | 搜索不存在关键词 | 返回空列表和 `Total=0` |

### Story E2-S2 · 文章保存和删除

**用户故事：** 作为用户，我希望编辑后能可靠保存文章，也能删除不需要的每日复盘，以便保持复盘库整洁。

**验收摘要：**
- AC1: 保存时 Markdown 文件是内容真源，SQLite 只作为索引。
- AC2: 删除每日复盘不删除图片，总复盘总结不可删除。

#### Issue E2-S2-I1 · 保存文章并同步 Front Matter 和索引

- **目标：** 实现 `SaveArticle(req)`，支持更新 Markdown 内容、标题、更新时间、图片数量和 SQLite 索引。
- **范围：** 校验 ID；读取已有索引；规范化 Front Matter；原子写入文件；扫描图片引用；索引失败时文章文件仍保存成功并返回 warning。
- **依赖：** E2-S1-I2、E1-S3-I1、E1-S3-I2。
- **备注/风险：** `Content` 可为空字符串但字段不能缺失；保存失败时必须保留编辑器内容由前端处理。

##### 验收标准
- AC1: 保存后文件内容更新，`updatedAt` 变化。
- AC2: 标题为空时可从 Front Matter 或一级标题提取。
- AC3: 数据库索引与文件元数据同步。

##### 功能测试
| 编号 | 用例名称 | 前置条件 | 输入/操作 | 预期结果 |
|------|----------|----------|-----------|----------|
| FT-1 | 保存正文更新 | 已有每日复盘 | 修改正文并调用保存 | 文件和详情均返回新内容 |
| FT-2 | 图片数量同步 | 正文含两个本地图片引用 | 调用保存 | `imageCount=2` |

##### 边界测试
| 编号 | 用例名称 | 前置条件 | 输入/操作 | 预期结果 |
|------|----------|----------|-----------|----------|
| BT-1 | 空正文保存 | 已有文章 | `Content=""` | 可保存，Front Matter 仍合法 |
| BT-2 | 损坏 Front Matter 保存 | 文章 Front Matter 被外部破坏 | 调用保存 | 正文保留，Front Matter 被重写为合法结构 |

#### Issue E2-S2-I2 · 删除每日复盘并处理孤立图片引用

- **目标：** 实现 `DeleteArticle(id)`。
- **范围：** 每日复盘删除 `.md` 文件和索引；总复盘总结拒绝删除；不删除 pictures 图片；相关资产 `article_id` 置空或标记孤立。
- **依赖：** E2-S1-I2。
- **备注/风险：** 图片清理器不属于首期范围，不应在删除文章时误删图片。

##### 验收标准
- AC1: 每日复盘删除后列表不再展示。
- AC2: 总复盘总结删除返回明确错误。
- AC3: 图片文件保留，资产索引解除文章关联。

##### 功能测试
| 编号 | 用例名称 | 前置条件 | 输入/操作 | 预期结果 |
|------|----------|----------|-----------|----------|
| FT-1 | 删除每日复盘 | 已有每日复盘 | 调用删除 | 文件和文章索引删除 |
| FT-2 | 图片不被删除 | 文章引用本地图片 | 删除文章 | 图片文件仍存在 |

##### 边界测试
| 编号 | 用例名称 | 前置条件 | 输入/操作 | 预期结果 |
|------|----------|----------|-----------|----------|
| BT-1 | 删除总结文章 | 总结文章存在 | 调用删除 | 返回“总复盘总结不可删除” |
| BT-2 | 删除不存在文章 | ID 不存在 | 调用删除 | 返回“文章不存在” |

### Story E2-S3 · 索引重建和 API 暴露

**用户故事：** 作为用户，我希望数据库损坏或文件被外部修改后仍可恢复文章列表，以便 Markdown 文件始终是可迁移真源。

**验收摘要：**
- AC1: 删除 `review.db` 后可通过扫描 `articles` 恢复文章列表。
- AC2: 前端可通过 Wails API 调用文章管理能力并收到明确错误。

#### Issue E2-S3-I1 · 重建文章索引

- **目标：** 实现 `RebuildIndex()`，扫描 `articles/*.md` 恢复 `review_articles`，并尽可能重连图片资产。
- **范围：** 解析 Front Matter；无 Front Matter 时按文件名推断每日复盘或总结；清空并重建文章索引；保留 `review_assets` 行；重新扫描图片引用。
- **依赖：** E2-S2-I1。
- **备注/风险：** 文件名不符合 `YYYY-MM-DD.md` 且不是 `复盘总结.md` 时需记录风险或跳过。

##### 验收标准
- AC1: 删除数据库或清空文章表后可恢复文章列表。
- AC2: 外部修改文章文件后重建索引能读取最新标题、摘要、标签、股票。
- AC3: 损坏文件不阻断其他文章重建。

##### 功能测试
| 编号 | 用例名称 | 前置条件 | 输入/操作 | 预期结果 |
|------|----------|----------|-----------|----------|
| FT-1 | 数据库恢复 | articles 下有两篇 Markdown | 清空 `review_articles` 后重建 | 两篇文章恢复到列表 |
| FT-2 | 外部修改同步 | 修改文件标题和标签 | 调用重建 | 列表显示新标题和标签 |

##### 边界测试
| 编号 | 用例名称 | 前置条件 | 输入/操作 | 预期结果 |
|------|----------|----------|-----------|----------|
| BT-1 | 非法文件名 | articles 下有 `abc.md` | 调用重建 | 跳过或记录 warning，不影响合法文章 |
| BT-2 | 单篇解析失败 | 一篇 Markdown Front Matter 损坏 | 调用重建 | 其他文章恢复，损坏文章按容错规则处理 |

#### Issue E2-S3-I2 · 暴露文章管理 Wails API

- **目标：** 在 `app.go` 中挂载 ReviewService 并暴露文章相关 API。
- **范围：** 增加 `reviewService` 字段；`NewApp()` 初始化服务；shutdown 关闭数据库；暴露 `GetReviewArticles`、`GetReviewArticle`、`CreateDailyReview`、`SaveReviewArticle`、`DeleteReviewArticle`、`GetReviewSummaryArticle`、`RebuildReviewIndex`。
- **依赖：** E2-S3-I1。
- **备注/风险：** 初始化失败不阻断主应用，但 API 必须返回明确错误或带 `Error` 字段的响应。

##### 验收标准
- AC1: Wails 可生成 Go 绑定，前端能调用复盘 API。
- AC2: ReviewService 未初始化时 API 不 panic。
- AC3: 删除和重建类接口返回 `"success"` 或明确错误文本。

##### 功能测试
| 编号 | 用例名称 | 前置条件 | 输入/操作 | 预期结果 |
|------|----------|----------|-----------|----------|
| FT-1 | API 创建并读取文章 | App 初始化成功 | 调用创建和读取 API | 返回文章详情 |
| FT-2 | API 重建索引 | App 初始化成功 | 调用 `RebuildReviewIndex` | 返回 `success` |

##### 边界测试
| 编号 | 用例名称 | 前置条件 | 输入/操作 | 预期结果 |
|------|----------|----------|-----------|----------|
| BT-1 | 服务初始化失败 | 模拟 `reviewService=nil` | 调用列表 API | 返回空结果或错误字段，不 panic |
| BT-2 | API 参数错误 | 传入非法日期 | 调用创建 API | 返回可展示错误 |
